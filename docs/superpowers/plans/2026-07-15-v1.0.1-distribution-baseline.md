# Sunset v1.0.1 Distribution Baseline Implementation Plan

**Date:** 2026-07-15

**Status:** Ready for execution

**Design spec:** `docs/superpowers/specs/2026-07-15-v1.0.1-distribution-baseline-design.md`

**Validated by:** `docs/superpowers/reviews/2026-07-15-v1.0.1-distribution-baseline-spec-rereview.md`

## Goal

Ship Sunset v1.0.1 from one GitHub Actions release authority with the canonical `github.com/enolalabs/sunset` identity, consistent version reporting for packaged and `go install` binaries, and five native archives that are packaged, extracted, smoke-tested, checksummed, staged as a draft, verified, approved, and only then published.

## Scope and execution rules

- Implement only the Phase 1 boundary from the design spec. Do not refactor parser, scanner, cache, engine, analyzer, or output behavior beyond shared version resolution.
- Preserve `PROJECT_AUDIT_REPORT.md` as an unrelated untracked user file; never stage it as part of these tasks.
- Use test-first changes for Go behavior and executable release helpers.
- Keep each repository task in its own commit so review and rollback remain narrow.
- Do not create `v1.0.1`, alter GitHub rulesets/environments, approve an environment, or publish a release until the explicit operator tasks at the end.
- A published v1.0.1 release is immutable. Any defect found after promotion requires v1.0.2.

5. Remove `main.Version` and `internal/output.Version`. Route CLI version/help output and `buildProjectInfo` through `version.Current()`.

6. Update `Makefile` release ldflags to target `internal/version.BuildVersion`. Local untagged builds may continue reporting `dev`.

## Task 3: Add native archive smoke-test helpers

**Files:**

- Create: `.github/scripts/release/smoke.sh`
- Create: `.github/scripts/release/smoke.ps1`
- Create: `.github/scripts/release/package-and-verify.sh`
- Create: `.github/scripts/release/package-and-verify.ps1`
- Create: `.github/scripts/release/verify-consumer.sh`

### Steps

1. Implement the POSIX smoke helper with arguments for executable, exact version, fixture, and temporary output directory. It must assert:

   - `version` exits zero and prints the exact version;
   - `languages` contains Go, JavaScript, TypeScript, and Python;
   - `parse --no-cache --quiet --output <dir> <fixture>` exits zero;
   - `<dir>/index.md` and at least one `<dir>/files/*.md` exist;
   - `index.md` contains the exact `sunset_version`.

2. Implement the PowerShell helper with the same contract and fail-fast semantics.

3. Implement POSIX packaging for Linux/macOS:

   - create exactly `sunset_<version>_<os>_<arch>.tar.gz`;
   - list the archive and require exactly one root entry named `sunset`;
   - extract into a new directory;
   - require the extracted file to be executable;
   - invoke `smoke.sh` against that extracted file;
   - return only the verified archive path.

4. Implement Windows packaging with `Compress-Archive`:

   - create exactly `sunset_<version>_windows_amd64.zip`;
   - expand into a new directory;
   - require exactly one root entry named `sunset.exe`;
   - invoke `smoke.ps1` against that extracted file;
   - return only the verified archive path.

5. Implement `verify-consumer.sh` to create a directory outside the checkout, initialize a temporary module, require the exact tag with `GOPROXY=direct`, compile and run a small program importing `github.com/enolalabs/sunset/pkg/sunset`, and require the four expected language names from `sunset.Languages()`.

6. Exercise the POSIX contract locally with a release-style Linux binary:

   ```bash
   tmp="$(mktemp -d)"
   go build -trimpath -ldflags "-s -w -X github.com/enolalabs/sunset/internal/version.BuildVersion=1.0.1" -o "$tmp/sunset" ./cmd/sunset
   bash .github/scripts/release/package-and-verify.sh "$tmp/sunset" 1.0.1 linux amd64 "$(pwd)/testdata/go-sample"
   tar -tzf sunset_1.0.1_linux_amd64.tar.gz
   ```

7. Run `bash -n` for all shell helpers. PowerShell syntax and runtime validation occurs on `windows-2025` in Task 4 because PowerShell is not installed locally.

8. Commit:

   ```bash
   git commit -m "test: add native release smoke contracts"
   ```

## Task 4: Replace the release workflow with the native staged state machine

**Files:**

- Replace: `.github/workflows/release.yml`
- Delete: `.goreleaser.yaml`
- Modify if required: `.gitignore`

### Workflow structure

Implement these jobs in dependency order:

1. `preflight`
2. `build_native` matrix, skipped for published mode
3. `stage_draft`, skipped for published mode
4. `verify_draft`, skipped for published mode
5. `promote`, skipped for published mode and protected by environment `release`
6. `verify_public`, used after promotion and as the only execution path for published-mode reruns

### Preflight requirements

- Trigger only on pushed `v*` tags.
- Set workflow default permission to `contents: read`; grant `contents: write` only to `stage_draft` and `promote`. Grant `actions: read` only where job timing queries require it.
- Add exact-tag concurrency with `cancel-in-progress: false`.
- Checkout with full history, pinned checkout SHA, and `persist-credentials: false`.
- Validate the exact tag with:

  ```text
  ^v(0|[1-9][0-9]*)\.(0|[1-9][0-9]*)\.(0|[1-9][0-9]*)$
  ```

- Derive and expose `tag`, version without `v`, and release mode.
- Fetch protected `main` and require `git merge-base --is-ancestor "$GITHUB_SHA" origin/main`.
- Run `go test ./... -race -count=1`, `go vet ./...`, `go mod tidy -diff`, and `go list -m` identity checks.
- Scan only active surfaces for `github.com/enolalab/sunset`, excluding historical review/spec/audit files.
- Query the exact tag with `gh release view` and expose `absent`, `draft`, or `published`.
- Record active job start/end/duration and write a concise job summary.

### Native matrix requirements

Use exactly:

| Target | Runner |
|---|---|
| `linux/amd64` | `ubuntu-24.04` |
| `linux/arm64` | `ubuntu-24.04-arm` |
| `darwin/amd64` | `macos-15-intel` |
| `darwin/arm64` | `macos-15` |
| `windows/amd64` | `windows-2025` |

- Set `fail-fast: false` and a 30-minute timeout.
- Assert `GOHOSTOS` and `GOHOSTARCH` equal the matrix target before building.
- Record runner image, `go version`, compiler path, and compiler version.
- On Windows, resolve preinstalled GCC using `Get-Command gcc`, set `CC` to the resolved executable, and require `CGO_ENABLED=1`.
- Build with `-trimpath -ldflags "-s -w -X github.com/enolalabs/sunset/internal/version.BuildVersion=<version>"`.
- Inspect architecture with `file` on Linux/macOS and MinGW `objdump -f` on Windows; require x86-64 or arm64 evidence matching the row.
- Package, extract, and smoke-test through the Task 3 helper for that OS.
- Upload exactly one archive using the pinned upload action. Name the internal artifact with tag and target and set a short explicit retention period such as 3 days.

### Draft staging requirements

- Use a 15-minute timeout and `contents: write`.
- Download internal archives with the pinned download action.
- Require the exact five filenames from the spec and reject duplicates/unexpected files.
- Compute sorted SHA-256 entries from final archive bytes into `checksums.txt`.
- For `absent`, create `Sunset v<version>` as a draft with curated notes from Task 5.
- For `draft`, remove stale exact-tag draft assets before uploading a fresh complete set; never operate on a public release.
- Upload exactly five archives and `checksums.txt` with `gh release upload`.

### Draft verification requirements

- Use a 20-minute timeout and read-only permissions.
- Checkout the exact tag with pinned checkout and no persisted credentials so `testdata/go-sample` exists.
- Download the exact draft asset set through the GitHub API into an empty directory; do not reuse internal artifacts.
- Verify every digest in `checksums.txt`.
- Use fresh `GOCACHE`, `GOMODCACHE`, and `GOBIN`.
- With `GOPROXY=direct`, install `github.com/enolalabs/sunset/cmd/sunset@<tag>` and run the CLI smoke helper against the absolute tagged fixture.
- Run `verify-consumer.sh` against the same exact tag.
- A failure leaves the release as draft.

### Promotion and public verification requirements

- `promote` uses a 10-minute execution timeout, `contents: write`, and `environment: release`. After approval, use `gh release edit <tag> --draft=false`; perform no checkout.
- `verify_public` uses a 20-minute timeout and read-only permissions. It must run after promotion for absent/draft modes and directly for published mode.
- Download all public assets through public release URLs into a new directory, require the exact set, and verify checksums.
- For published-mode reruns, repeat exact-tag checkout, clean-cache CLI installation smoke, and external consumer-module verification. Never mutate release data.
- Query job timing data and summarize mode, each stage, every matrix duration, total job-minutes, wall-clock time, excluded queue/approval time, and active critical path. Compare only absent-mode critical path with the non-blocking 30-minute target.

## Task 8: Execute and verify the v1.0.1 release

**External state — tag creation and promotion require explicit maintainer authorization.**

1. Confirm Tasks 1–7 are complete on protected `main`, the worktree contains no unintended changes, and no v1.0.1 release/tag already exists unexpectedly.

2. Create and push annotated tag `v1.0.1` from the approved commit.

3. Observe preflight and all five native jobs. Require host/toolchain evidence and post-extraction smoke success for every target.

4. Inspect the draft release before approval. Require exactly:

   - `sunset_1.0.1_linux_amd64.tar.gz`
   - `sunset_1.0.1_linux_arm64.tar.gz`
   - `sunset_1.0.1_darwin_amd64.tar.gz`
   - `sunset_1.0.1_darwin_arm64.tar.gz`
   - `sunset_1.0.1_windows_amd64.zip`
   - `checksums.txt`

5. Require draft checksum verification, clean-cache `go install`, CLI smoke, generated metadata version, and external consumer-module test to pass.

6. A designated maintainer approves the `release` environment. The workflow promotes the existing draft without replacing assets.

7. Require anonymous public download/checksum verification to pass and inspect the workflow timing summary. A target miss above 30 minutes is a warning, not a v1.0.1 blocker.

8. Download one suitable archive independently, verify its checksum with the documented user command, extract it, and run `sunset version`.

9. If any defect appears after publication, leave v1.0.1 immutable and open a v1.0.2 corrective plan.

## Definition of done

- Canonical module/import/install identity is `github.com/enolalabs/sunset` on all active surfaces.
- Packaged and `go install` binaries report `1.0.1` consistently in CLI and generated metadata.
- Five exact native archives pass post-extraction smoke tests on their matching OS/architecture runners.
- One sorted SHA-256 checksum file matches the public bytes.
- Draft verification and maintainer approval precede publication.
- Public releases are immutable and published-mode reruns are read-only.
- External library consumption from the exact tag compiles and runs.
- README, release notes, and operator documentation describe the real support and trust boundaries.
- Full tests, race detector, vet, tidy diff, actionlint, identity scan, and public verification pass.
