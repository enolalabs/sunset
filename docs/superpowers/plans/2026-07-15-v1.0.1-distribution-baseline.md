# Sunset v1.0.1 Distribution Baseline Implementation Plan

**Date:** 2026-07-15

**Status:** Ready for execution

**Design spec:** `docs/superpowers/specs/2026-07-15-v1.0.1-distribution-baseline-design.md`

**Validated by:** `docs/superpowers/reviews/2026-07-15-v1.0.1-distribution-baseline-spec-rereview.md`

## Goal

Ship Sunset v1.0.1 from one GitHub Actions release authority with the canonical `github.com/enolalabs/sunset` identity, consistent version reporting for packaged and `go install` binaries, and five native archives that are packaged, extracted, smoke-tested, checksummed, staged as a draft, verified, approved, and only then published.

## Scope and execution rules

- Implement only the Phase 1 boundary from the design spec. Do not refactor parser, scanner, cache, engine, analyzer, or output behavior beyond shared version resolution.
- Preserve `PROJECT_AUDIT_REPORT.md` as an unrelated untracked user file; never stage it as part of these tasks.
- Use test-first changes for Go behavior and executable release helpers.
- Keep each repository task in its own commit so review and rollback remain narrow.
- Do not create `v1.0.1`, alter GitHub rulesets/environments, approve an environment, or publish a release until the explicit operator tasks at the end.
- A published v1.0.1 release is immutable. Any defect found after promotion requires v1.0.2.

**Files:**

### Helper interfaces

- `smoke.sh <executable> <version> <fixture> <output-dir>` writes diagnostics only to stderr and exits zero without stdout on success.
- `smoke.ps1 -Executable <path> -Version <version> -Fixture <path> -OutputDir <path>` has the same contract.
- `package-and-verify.sh <executable> <version> <os> <arch> <fixture> <archive-output-dir>` writes exactly one absolute verified archive path to stdout.
- `package-and-verify.ps1 -Executable <path> -Version <version> -Fixture <path> -ArchiveOutputDir <path>` writes exactly one absolute verified archive path to stdout.
- `verify-consumer.sh <tag> <work-dir>` requires the caller to provide already-exported cache variables, creates the external module below `work-dir`, and exits zero without stdout only after compile/run assertions pass.
- Every helper rejects missing/extra arguments, writes actionable failures to stderr, and returns non-zero without leaving a success path or archive on failure.

### Steps

1. Before implementing the helpers, write fixture-driven tests in `test-helpers.sh` and `test-helpers.ps1`. Cover the successful smoke/package path plus wrong version, missing language, missing `index.md`, no per-file output, malformed archive, extra root entry, wrong executable name, lost POSIX execute bit, and consumer compile/language mismatch. Tests must demonstrate at least one expected failure before implementation.

2. Implement the POSIX and PowerShell smoke helpers. Both must assert exact version output, all four languages, flag-before-fixture parsing, `index.md`, at least one `files/*.md`, and exact `sunset_version`.

3. Implement POSIX packaging for Linux/macOS. Create the exact archive in the requested output directory, require one root `sunset` entry, extract into a new directory, require executable permission, run the smoke helper against the extracted file, and print only the verified archive path.

4. Implement Windows packaging with `Compress-Archive`. Create the exact zip in the requested output directory, require one root `sunset.exe`, extract into a new directory, run the PowerShell smoke helper, and print only the verified archive path.

5. Implement `verify-consumer.sh` with `GOPROXY=direct`: initialize a module outside the checkout, require the exact tag, compile/run a program importing `github.com/enolalabs/sunset/pkg/sunset`, and require Go, JavaScript, TypeScript, and Python from `sunset.Languages()`.

6. Run POSIX tests and a release-style Linux cycle entirely below a temporary directory:

   ```bash
   bash -n .github/scripts/release/*.sh
   bash .github/scripts/release/test-helpers.sh
   tmp="$(mktemp -d)"
   go build -trimpath -ldflags "-s -w -X github.com/enolalabs/sunset/internal/version.BuildVersion=1.0.1" -o "$tmp/sunset" ./cmd/sunset
   bash .github/scripts/release/package-and-verify.sh "$tmp/sunset" 1.0.1 linux amd64 "$(pwd)/testdata/go-sample" "$tmp/archives"
   ```

7. The PowerShell test suite must run successfully on `windows-2025` in the read-only pre-tag validation workflow in Task 5. Do not claim PowerShell runtime validation from local syntax checks.

8. Commit:

   ```bash
   git commit -m "test: add native release smoke contracts"
   ```

### Read-only pre-tag validation workflow

1. `release-validation.yml` uses `workflow_dispatch` with required 40-hex `commit_sha`, default `contents: read`, no write permission, no release mutation, and concurrency keyed by that commit.
2. `validate_preflight` has `timeout-minutes: 30`, checks out that exact commit with full history, pinned checkout, and `persist-credentials: false`; requires the commit to be reachable from protected `main`; uses pinned setup-go with `go-version-file: go.mod`; and runs test/race, vet, tidy diff, module identity, active-surface identity, helper tests, snippet consistency, and workflow-policy tests.
3. `validate_native` runs the exact five-row native matrix below with `fail-fast: false`, packages/extracts/smokes each archive, and runs `test-helpers.ps1` on Windows. It is read-only and uploads evidence artifacts only.
4. Each native row creates a one-archive `checksums.txt`, serves its verified archive and checksum over an ephemeral loopback HTTP endpoint, and executes the matching canonical Task 4 installation snippet without changing its checksum/extraction/version commands.
5. The workflow summary records input commit, protected-main reachability, five host/toolchain identities, helper results, snippet results, archive names, and checksums. Task 7 records the successful run URL as release evidence.

### Release workflow structure and exact DAG

Implement `preflight`, `build_native`, `stage_draft`, `verify_draft`, `promote`, `verify_public`, and `telemetry`.

- `build_native` needs successful `preflight` and runs only when mode is not `published`.
- `stage_draft` needs successful `preflight` and `build_native`, and runs only when mode is `absent` or `draft`.
- `verify_draft` needs successful `preflight` and `stage_draft`, and runs only when mode is `absent` or `draft`.
- `promote` needs successful `preflight` and `verify_draft`, runs only when mode is `absent` or `draft`, and uses environment `release`.
- `verify_public` needs `preflight` and `promote`. Its `if: always()` condition must additionally require `preflight == success` and either mode `published` with `promote == skipped`, or mode `absent/draft` with `promote == success`.
- `telemetry` needs every prior job, uses `if: always()` after preflight, is read-only, and never changes release state.
- Add fixture-based truth-table tests for absent, draft, published, and every upstream failure/skip outcome; `always()` may bypass skip propagation only with explicit result checks.

### Preflight and release-state contract

- Trigger only on pushed `v*` tags. Add exact-tag concurrency with `cancel-in-progress: false` and `timeout-minutes: 30`.
- Use default `contents: read`. Every checkout uses the pinned SHA, full history when required, and `persist-credentials: false`.
- Use pinned setup-go with `go-version-file: go.mod` in every job that runs Go.
- Validate the exact SemVer regex, peel the annotated tag, require its commit to equal the approved commit and be reachable from protected `main`, and expose tag, version, peeled commit, mode, release ID, draft flag, and target commit.
- Run race tests, vet, tidy diff, module identity, active-surface identity, and release-note/snippet policy checks.
- For each `gh` command, set step-scoped `GH_TOKEN: ${{ github.token }}` with the matching minimum job permission. Never expose it to build, helper, or smoke steps.
- Treat only an authoritative GitHub not-found response as `absent`. Authentication, authorization, rate-limit, network, JSON/parse, or inconsistent-state errors fail closed.
- For an existing release, record and validate its exact ID, tag, draft state, and target. Published mode disables every write-capable job.
- Upload the normalized release-notes file and its SHA-256 digest from this read-only exact-tag checkout as a bounded internal artifact for `stage_draft`; do not interpolate note contents into shell source.

### Native matrix contract

Use exactly Linux amd64 on `ubuntu-24.04`, Linux arm64 on `ubuntu-24.04-arm`, Darwin amd64 on `macos-15-intel`, Darwin arm64 on `macos-15`, and Windows amd64 on `windows-2025`.

- Set `fail-fast: false`, `timeout-minutes: 30`, pinned setup-go with `go-version-file: go.mod`, and `CGO_ENABLED=1` on all five rows; assert GOHOSTOS, GOHOSTARCH, and CGO state before build.
- Record runner image, `go version`, `gh version`, compiler path/version, and architecture evidence. Resolve/record both `gcc` and `objdump` on Windows.
- Build with release ldflags, package/extract/smoke through Task 3, and upload exactly one tag/target-named archive with `retention-days: 3`.

### Draft staging, verification, and promotion

- `stage_draft` has `timeout-minutes: 15` and `contents: write`. Immediately re-query state: require no release in absent mode or the same recorded release ID still in draft mode; abort on published state or identity mismatch. Delete assets only from that verified draft ID, then upload a fresh exact six-file set. A partial failure leaves a recoverable draft and the runbook permits only a draft rerun.
- Use the notes artifact from preflight with `gh release create --notes-file <path>`; verify its recorded digest and never checkout or interpolate Markdown in the write-capable job.
- `verify_draft` has `timeout-minutes: 20`, read-only permissions, exact-tag checkout without persisted credentials, and pinned setup-go `cache: false`. Before any Go command, create/export unique `GOCACHE`, `GOMODCACHE`, and `GOBIN`; the consumer check shares only these caches that were empty at job start.
- Download exact draft assets through the API, verify checksums, run exact-tag CLI smoke and external-consumer verification, and create `draft-evidence.json` containing release ID, peeled commit, normalized notes digest, exact asset names/sizes/digests, and checksum-file digest. Upload this bounded evidence artifact.
- `promote` has `timeout-minutes: 10`, `contents: write`, and protected environment `release`. After approval, download the evidence artifact, re-query the same draft ID, reassert the immutable tag binding/target/notes digest, re-download all six assets, and verify names, sizes, digests, and checksum bytes before `gh release edit <tag> --draft=false`. Perform no checkout.

### Public verification and telemetry

- `verify_public` has `timeout-minutes: 20` and read-only permissions. For absent/draft mode, download all six assets anonymously without `Authorization` or `GH_TOKEN`, require the exact set, and verify checksums. For published mode, also perform exact-tag checkout, setup-go with `cache: false`, unique caches exported before Go commands, CLI install/smoke, and consumer verification. Never mutate release data.
- `telemetry` has `timeout-minutes: 10`, `actions: read`, and step-scoped `GH_TOKEN` only for paginated API queries. It runs after `verify_public`, so all measured jobs have final timestamps; it excludes itself from metrics.
- `collect-timing.sh` must define job duration as `completed_at - started_at`, total job-minutes as the sum of non-skipped measured jobs, matrix duration as the longest native row, absent active critical path as sequential stages plus that matrix maximum, and published active path as preflight plus public verification. Draft/published modes are reported separately and never compared with the absent target.
- Wall-clock is telemetry observation time minus workflow creation time. Report `wall-clock - active-critical-path` as a combined queue/approval/orchestration residual rather than claiming an unavailable exact split. Missing required timing data fails only the telemetry job and cannot trigger mutation; an absent-mode critical path above 30 minutes emits a warning only.
- Test timing logic with fixture JSON for absent, draft, published, skipped, failed, paginated, and null-timestamp cases.

### Workflow policy and validation

1. `verify-workflow-policy.sh` parses both workflow files. Each non-local `uses:` entry must use exactly one of the four allowlisted official actions at its listed 40-hex SHA; only `./` local actions are otherwise permitted. It also requires `persist-credentials: false` in every checkout block, exact finite job timeouts, and no `softprops`/GoReleaser reference.
2. Write failing policy fixtures for `@stable`, branch refs, short SHAs, unknown actions, missing checkout isolation, missing timeout, and write permission in the validation workflow before implementing the policy helper.
3. Validate both workflows with pinned `actionlint@v1.7.7`, run workflow-policy/timing/helper tests, repository tests, and shell syntax checks.
4. Commit:

## Definition of done

- Canonical module/import/install identity is `github.com/enolalabs/sunset` on all active surfaces.
- Packaged and `go install` binaries report `1.0.1` consistently in CLI and generated metadata.
- Five exact native archives pass post-extraction smoke tests on their matching OS/architecture runners.
- A read-only pre-tag workflow validates the exact protected-main commit on all five native targets before `v1.0.1` is created.
- One sorted SHA-256 checksum file matches the public bytes.
- Draft verification and maintainer approval precede publication.
- Public releases are immutable and published-mode reruns are read-only.
- External library consumption from the exact tag compiles and runs.
- README, release notes, and operator documentation describe the real support and trust boundaries.
- The canonical Linux, macOS, and Windows install snippets pass on matching native validation runners.
- Full tests, race detector, vet, tidy diff, actionlint, identity scan, and public verification pass.
